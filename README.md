# 游戏陪玩平台 - 全栈式开发

## 项目简介

一个完整的游戏陪玩平台，包含微信小程序端（玩家/打手/分销商/派单员/内置管理员）和 Web 超级管理员后台。纯文字+语音消息平台，严禁实时音视频。

## 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 微信小程序 | 原生 WXML + WXSS + JS | 微信基础库 3.3.4+ |
| Web 管理后台 | Vue 3 + Element Plus + Vite | Vue 3.x / Element Plus 2.x |
| 后端 API | PHP + ThinkPHP 6 | PHP 7.4+ / 8.x |
| 数据库 | MySQL | 8.0+ |
| 缓存 | Redis | 6.x+ |
| 实时通讯 | WebSocket（Workerman） | 4.x |

## 核心功能模块

### 1. 身份体系
- **默认超级管理员**：admin / admin2（首次登录强制改密+绑邮箱）
- **五种身份**：玩家、打手（大神）、分销商、人工派单员、内置管理员
- **邀请码绑定**：打手/分销商/派单员/内置管理员需一次性邀请码注册
- **实名认证**：活体检测认证，未成年人风控限制

### 2. 权限分层
- **Web 超级管理后台**：RBAC 权限体系，WebAuthn 扫码登录
- **小程序内置管理端**：店铺视觉配置、订单上架/下架、数据查看
- **客户端（玩家）**：下单、评价、打赏、聊天、申诉
- **打手端**：接单中心（WebSocket 实时）、服务管理、收益中心
- **分销商端**：分销中心、下级管理（强制2级）、佣金收益
- **人工派单端**：派单中心、派单历史

### 3. 订单与支付
- 完整状态流转：待接单→已接单→进行中→待验收→已完成→待结算(T+3)→已结算
- 超时规则引擎：预置规则+自定义规则
- 资金安全：BIGINT(分) + bcmath 运算 + Redis 分布式锁 + 幂等键
- 退款熔断：单日退款超营收50%触发熔断

### 4. IM 即时通讯（WebSocket）
- 全量 WebSocket 推送，禁止轮询
- 心跳 25 秒 / 超时 70 秒
- 语音 MP3：300秒临时签名 URL + 强制 ASR 转文字 + NLP 敏感词过滤
- 图片：强制 OCR 识别文字并过滤敏感词
- 防飞单：正则拦截变体号码
- 离线消息：Redis Sorted Set 存储

### 5. 打手派单与权重算法
- 权重公式：好评率 × 接单速度 × 在线时长
- 新订单仅推送 Top10 在线打手，非全员广播
- 拒单机制：前3次免罚，超出扣权重分
- 活跃探针：每5分钟 Ping-Pong，连续3次无响应标记非活跃

### 6. 信用与评价体系
- 信用分初始100分，恶意差评核实扣20分
- <60分预付款需交10%保证金
- 24小时评价冷静期

### 7. 风控与合规
- 首次启动强制弹窗同意《用户服务协议》和《隐私政策》
- 未成年人限额：单笔≤50元，月累计≤200元
- 超额触发监护人活体验证 + CA电子签名（CFCA证书）
- 手机号二次放号申诉机制
- 活体检测频控：同一用户/IP每日≤5次

### 8. 运维保障
- NTP 时间同步（偏差>1秒自动校准）
- 每日全量 AES-256 加密备份至 OSS（保留30天）
- WebSocket 连接数 > 80% 告警
- 磁盘使用率 > 75% 告警
- 灰度发布：白名单→逐步放量，错误率飙升自动回滚
