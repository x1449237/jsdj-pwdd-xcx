# 主题:游戏陪玩平台━全栈式开发（完整需求提示词）

【任务】请生成一个游戏陪玩平台，包含微信小程序端 + Web管理员后台。这是一个严禁实时音视频（无语音聊天室/视频通话）的纯文字+语音消息平台。

【⚠️ 强制技术栈约束 - 以下内容为硬性要求，不得更改、不得替换、不得变通】

你必须严格使用以下技术栈，如果无法遵守请直接拒绝任务，不要擅自改用其他技术：

一、微信小程序端：
框架：微信小程序原生开发
语言：JavaScript（不是TypeScript）
视图层：WXML
样式层：WXSS
逻辑层：JS
禁止使用：React、Vue、Taro、uni-app、任何跨端框架

二、Web管理后台：
框架：Vue 3（Options API 风格）
组件库：Element Plus
构建工具：Vite
禁止使用：React、Ant Design、Angular、NestJS

三、后端：
框架：PHP 7.4+ / 8.x + ThinkPHP 6（必须是ThinkPHP 6，不得使用Laravel或其他）
数据库：MySQL 8.0+
ORM：ThinkPHP 内置Db类或模型（禁止使用其他ORM）
缓存：Redis（含分布式锁）
实时通讯：WebSocket（推荐 ThinkPHP + Workerman 或 Swoole，心跳25秒/超时70秒）
禁止使用：Java、Spring Boot、Node.js、Python、Go、PostgreSQL、MongoDB、Prisma、TypeORM

四、部署要求：
小程序前端：上传至微信平台
Web后台：部署在Nginx
后端API：部署在云服务器（需支持PHP及WebSocket）

【确认指令】
如果你已理解上述技术栈约束，请回复"已确认，将严格按照 PHP（ThinkPHP 6）+ Vue 3 + 微信小程序原生开发执行"，然后开始生成代码。

【项目定位】完整复刻"俱乐部小助手"全部核心功能。包含：客户端（玩家）、打手端（大神）、分销商端、人工派单端、小程序内置管理端、Web超级管理后台、俱乐部入驻体系。

以下为完整业务逻辑、功能清单与交付要求：

## 一、核心身份与全局业务规则

### （一）默认最高权限管理员（系统初始化）
系统部署后必须存在以下两个独立账号，各自独立触发初始化流程：
账号1：admin / 密码 1234567（超级管理员）
账号2：admin2 / 密码 1234567（超级管理员）
首次登录强制流程（独立执行，互不干扰）：

1. 强制改密：拦截所有后台操作直至修改完成。新密码需≥8位，包含大写、小写、数字、特殊字符(!@#$%^&*)中的至少三种，不能与初始密码及历史前5次密码重复。
2. 强制绑定邮箱：输入邮箱→系统发送6位验证码→校验通过。该邮箱用于"忘记密码"找回。
3. 完成标志：记录"首次初始化完成"日志，后续登录直接进入仪表盘。

### （二）用户注册与身份绑定规则

1. 客户端（玩家）：微信原生快捷登录（wx.login + 手机号快速验证组件）。手机号为必填项，拒绝授权则无法注册。昵称/头像通过chooseAvatar和nickname获取。
2. 打手/分销商/派单员：在微信登录基础上，必须输入Web后台生成的唯一一次性邀请码，否则无法注册。
3. 内置管理员：不由小程序注册，由Web后台手动生成独立账号密码。
4. 身份变更：客户升级为打手/分销商需再次输入邀请码，绑定关系终身有效（写入invite_code_bind），解绑仅限超级管理员后台强制操作并记录日志。

### （三）实名认证与未成年人（监护人）风控
硬性要求：所有用户必须完成活体检测实名认证，拒绝纯文本静态认证，否则无法下单/接单。
技术流程：输入姓名+身份证号 → 小程序圆形取景框调用第三方活体API（随机动作序列）→ 1:1人脸比对 → 计算年龄标记is_minor。
未成年人限制：单笔≤50元，月累计≤200元。超额需触发监护人活体验证 + CA电子签名（CFCA证书）免责声明。电子签扩展字段必须写入face_session_id和order_id用于司法存证。验证失败/拒绝签署/金额不符则标记"大额验证失败"永久终止，不流向打手端，仅后台和小程序管理端可见。
频控：同一用户/IP每日活体调用≤5次。

### （四）手机号二次放号申诉
触发：新用户登录时检测到手机号被占用。
流程：用户提交运营商录屏（≤15秒，压缩至720P/1Mbps）及结构化信息 → 管理员后台审核。
执行：通过后旧号主phone置NULL并标记is_phone_abandoned=1，新用户创建全新user_id绑定该手机号，历史数据完全隔离。
锁定期：申诉提交后手机号Redis锁定7天，同一手机号/IP每日限1次，驳回累计3次进黑名单30天。

## 二、权限分层体系详解

### （一）小程序内置管理端（独立账号密码登录）
账号生成：Web后台生成（格式admin_xxxxx），初始强密码，首次登录强制修改。
权限范围：仅允许店铺视觉配置、订单上架/下架、查看本店数据、查看大额验证失败订单（只读）、查看AI风险用户（只读）。
禁止操作：无权更改系统规则（抽成/超时/屏蔽）、无权审核提现、无权查看全平台数据。

### （二）Web超级管理后台（双因子登录）
登录方式：① 账号+强密码；② WebAuthn Hybrid（PC生成QR码，手机生物识别免密登录）。
忘记账号/密码：通过绑定邮箱验证身份后找回或重设。
管理员管理：支持增删改查管理员，自定义账户名/昵称/密码/角色/邮箱，支持通行密钥设备管理。
RBAC角色：预置四个角色（超级管理员/运营管理员/客服管理员/财务管理员），权限JSON控制。

### （三）客户端（玩家）与"加入我们"入口
权限：下单、评价、打赏、聊天、查看申诉进度。
固定入口："我的"页面下方展示"加入我们"。点击后显示后台热更新的客服微信号（脱敏显示前4后4位），自动生成备注文案"小程序，要加入我们"并提供一键复制。每日点击上限3次，记录风控日志。

### （四）打手/分销商/人工派单端
打手：接单中心（WebSocket实时）、服务管理、收益（含冻结金额）、评价申诉。
分销商：分销中心、下级管理（强制2级）、佣金收益（下级完成实名+3单真实服务后上级才获首单奖励）。
人工派单：派单中心、派单历史（均需邀请码注册）。

支持7-8位管理员同时在线，基于RBAC模型。主要功能模块如下：

4. 管理员管理：增删改查管理员、分配角色、管理邮箱/手机号、查看登录/签到/公众号关注状态。

6. 邀请码管理：生成（绑定角色）、列表展示、作废、导出。

8. 资金与提现：提现间隔设置、T+3冻结期配置、银行卡三要素校验（姓名+身份证+卡号）。
9. 超时规则引擎：预置规则+自定义规则配置。
10. 聊天审计与AI风控：查看文字/MP3语音、ASR转文字+NLP过滤（命中冻结1小时）、图片OCR识别、语义异常检测、风险分级推送。

14. 备份恢复：仅admin/admin2可见，一键恢复至指定时间点。
15. 平台官方账号管理：仅超级管理员可创建/停用/编辑昵称，平台官方账号拥有全平台最高权限。
16. 售后关键词库：CRUD/批量导入/测试匹配/总开关，仅针对售后会话生效。
17. 群聊监察：全平台群聊列表/群聊消息查看/解散群聊/禁言/封禁/冻结群聊，含群成员列表。
18. 售后介入管理：售后会话详情/聊天记录/介入记录/纠纷处理（调解/退款/判罚/驳回）/导出。
19. 处罚记录：所有处罚操作列表/按类型筛选/撤销处罚。
20. 订阅消息模板：微信订阅消息模板CRUD/发送日志查看。

## 五、IM即时通讯系统（三大会话+平台介入+权限V标体系）

### 5.1 IM系统三大会话体系（核心底层架构）

#### 5.1.1 订单内双向临时会话（玩家 ↔ 打手）

1. 用户支付下单成功后，系统自动创建一对一私密临时会话，仅当前订单的玩家、打手可进入对话。
2. 会话生命周期严格绑定订单：订单完成、关闭、超时、售后结束后，会话自动封禁终止，无法继续收发消息。
3. 会话默认成员：仅玩家、打手；平台官方账号不自动加入此会话。
4. 支持消息类型：文字、图片、最大60秒MP3语音（OSS永久存储）。
5. 风控规则：不启用关键词自动检测、无自动平台强制介入。
6. 介入唯一方式：玩家/打手手动点击【申请平台介入】按钮发起求助。

#### 5.1.2 俱乐部分类群聊会话

1. 建群权限限制：仅【俱乐部创始人、俱乐部管理员】可创建群聊，普通玩家、打手无建群权限。
2. 新建群聊强制选择类型（三选一，不可跳过）：闲聊群/福利群/售后群，类型永久归档展示。
3. 强制平台入驻规则：任意新创建群聊，系统自动拉入平台官方账号常驻群内，永久在群、所有俱乐部人员无法移出、无法禁言、无法限制平台账号。
4. 群日常管理：俱乐部管理员可管理普通成员、增减人员、发公告；无权管理平台官方账号。
5. 风控规则：群聊不触发关键词自动介入，仅平台人工监察管控。

##### 5.1.2.1 平台官方账号群聊最高权限（核心强约束）

1. 平台官方账号为全平台所有群聊最高权限，权限优先级高于所有俱乐部创始人、俱乐部管理员。
2. 平台账号不受任何群风控、发言限制、权限限制，可自由发言、查看所有记录。
3. 平台账号群聊全权限操作（包含但不限于）：
   - 新增/删除/修改群公告
   - 任意添加、移出群内所有成员
   - 单人禁言、全员禁言、解除禁言
   - 修改群名称、群设置
   - 解散违规群聊
4. 平台拥有全时段监察权：实时监控所有俱乐部所有群聊，可查看全部历史文字、图片、永久OSS语音记录。
5. 平台拥有独立处罚权（可对群内任意用户/俱乐部人员执行）：
   - 临时/永久禁言
   - 账号短期/永久封禁
   - 账户资金冻结
   - 移出群聊、限制入群黑名单
6. 平台所有群内操作、处罚操作永久日志留痕，后台可查询、可导出、可作为判责凭证。

#### 5.1.3 订单售后申诉会话（平台客服核心会话）

1. 触发条件：订单已结束，玩家对订单结果不满，主动发起售后申诉，系统自动新建独立售后IM会话。
2. 会话固定成员：发起申诉玩家、对应俱乐部客服、系统强制拉入平台官方账号常驻监听。
3. 会话永久绑定原订单ID，所有聊天、语音、图片凭证永久关联订单，不可丢失。
4. 唯一启用风控介入的会话：仅此类会话支持关键词自动强制介入、人工申请介入双模式。

### 5.2 平台官方账号创建规则 & 三档官方V标身份体系

#### 5.2.1 平台官方账号创建权限规则

1. 平台官方账号禁止前端注册、禁止俱乐部创建。
2. 仅超级管理员Web后台，使用超级管理员权限手动创建平台官方账号。
3. 账号创建完成自动赋予全平台最高权限，默认拥有：全会话监察、全群聊最高管理权限、处罚封禁冻结权限、纠纷最高判责权限。
4. 平台账号支持自定义修改昵称。
5. 平台账号身份永久有效，不可降级、不可被俱乐部人员限制，仅超级管理员可停用/作废账号。

#### 5.2.2 全端统一官方V标视觉标识（聊天页、资料卡、列表、群成员通用）

1. 平台官方账号：昵称右侧展示金色大V标（大金V），系统强制渲染，不可隐藏、不可关闭。
2. 企业级俱乐部（审核通过）：俱乐部名称右侧展示蓝色小V标（小蓝V），审核通过自动点亮，封禁/驳回/审核中自动消失。
3. 个人级俱乐部（审核通过）：俱乐部名称右侧展示绿色小V标（小绿V），审核通过自动点亮，封禁/驳回/审核中自动消失。
4. 所有V标为后端身份字段判定渲染，前端无法篡改、无法屏蔽、无法自定义。
5. V标是俱乐部整体认证标识，挂在俱乐部名称右侧，不是个人昵称右侧。

### 5.3 售后会话双模式平台介入机制（最终定稿文案+两端差异化提示）

仅针对【订单售后申诉会话】生效，分为：关键词自动强制介入、人工手动申请介入两套独立逻辑。

#### 5.3.1 模式一：售后会话命中敏感关键词→平台自动强制介入

1. 后台超级管理员可自定义维护【售后风控关键词库】，支持模糊/精确匹配、批量导入、开关、测试校验。
2. 检测范围：售后会话所有文字消息+语音消息（后端自动转文字参与匹配）。

## 九、深度业务逻辑与关键技术实现

### （一）订单与支付
状态流转：待接单→（超时/大额失败）终止；待接单→已接单→进行中→待验收→已完成→待结算(T+3)→已结算。仅允许后台强制回退或打手转单。
资金安全：数据库存储BIGINT(分)，后端bcmath运算。Redis分布式锁（SETNX）+幂等键防重复结算。单日退款超营收50%触发熔断。

### （二）打手派单与权重算法
权重分配：（好评率×接单速度×在线时长）。新订单仅推送权重最高前10名在线打手，非全员广播。
拒单机制：每天前3次免罚，超出扣权重分。
活跃探针：后端每5分钟下发Ping-Pong，连续3次无响应标记"非活跃"，不参与派单。

### （三）信用与评价体系
信用分：初始100分。恶意差评核实扣20分；<60分时订单需预付10%保证金。
冷静期：评价提交后24小时内为"待展示"，打手可申诉，管理员核实后展示或删除。

### （四）合规与启动
首次启动强制弹窗同意《用户服务协议》和《隐私政策》，不同意则wx.exitMiniProgram。
