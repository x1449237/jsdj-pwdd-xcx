# 《戟三电竞》平台优化升级全方案（V2.0 完整版）

## 项目概述

本项目是仿制【俱乐部小助手app】的游戏陪玩平台，小程序名称为【戟三电竞】。在原有核心功能基础上，新增9大模块，覆盖资金合规、风控安全、订单履约、俱乐部运营、营销裂变、IM通讯、后台管理、技术架构、纠纷仲裁等维度。

### 技术栈（强制约束）

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | ThinkPHP | 6.x |
| 编程语言 | PHP | 7.4+/8.x |
| 实时通讯 | Swoole | 4.x（替代原Workerman） |

## 一、资金结算合规体系

### 1.1 分账规则引擎

- **四角色分账**：打手、俱乐部、分销商、平台，比例独立配置，总和校验100%
- **阶梯分润**：后台可视化配置，不受微信原生30%分账上限约束
- **默认规则**：支持设置默认分账规则，新订单自动应用

### 1.2 退款反向分账

- **自动回滚**：订单退款时按原分账比例反向追回资金
- **坏账闭环**：从各角色待结算余额扣减，杜绝坏账
- **回滚记录**：生成完整退款分账凭证，支持审计追溯

### 1.3 个税代扣

- **税率配置**：后台配置不同角色税率和起征点
- **自动扣除**：提现时自动计算并扣除个税
- **完税凭证**：生成个税代扣记录，满足合规要求

### 1.4 子商户账户

- **独立结算**：打手、俱乐部可开通独立结算账户
- **双渠道提现**：微信自动转账 + 银行卡打款
- **批量操作**：支持批量审核、批量打款

### 1.5 数据库表

| 表名 | 说明 |
|------|------|
| `profit_share_rule` | 分账规则表 |
| `profit_share_record` | 分账记录表 |
| `profit_share_refund` | 退款反向分账表 |
| `tax_config` | 个税配置表 |
| `tax_record` | 个税代扣记录 |
| `merchant_account` | 子商户账户表 |
| `withdraw_batch` | 提现批次表 |

## 二、未成年人风控体系升级

### 2.1 宵禁拦截

- **时间范围**：22:00 - 次日08:00
- **拦截操作**：未成年人无法下单、支付、打赏、进群
- **前端提示**：弹窗提示监护时段限制
- **后台日志**：记录拦截日志，支持统计分析

### 2.2 大额消费预警

- **三级阈值**：
  - 80%：推送监护人订阅消息提醒
  - 100%：需监护人二次人脸验证
  - 超额：强制阻止并记录台账
- **月度额度**：默认200元，后台可配置

### 2.3 活体缓存优化

- **缓存周期**：7天内重复操作无需重复活体
- **频控保留**：每日5次调用限制不变
- **成本降低**：减少第三方接口调用成本

### 2.4 家长监护后台（小程序内嵌）

监护人绑定子女账号后可：
- 查看消费账单、订单记录、聊天记录摘要
- 设置月度消费限额
- 一键冻结子女账号
- 关闭打赏/下单功能
- 查看月度消费报告

### 2.5 实名前置拦截

- **强制校验**：下单、进群聊、打赏、申请俱乐部入驻前强制校验实名状态
- **未完成跳转**：未实名直接跳转认证页，避免无效订单

### 2.6 数据库表

| 表名 | 说明 |
|------|------|
| `minor_curfew_log` | 宵禁拦截日志 |
| `minor_consume_warning` | 消费预警记录 |
| `parent_guardian_bind` | 家长监护绑定 |
| `parent_guardian_setting` | 监护设置 |
| `parent_consume_report` | 消费账单月报 |

## 三、订单体系深度优化

### 3.1 多订单类型

| 类型 | 说明 | 状态流转 |
|------|------|----------|
| 即时单 | 立即下单匹配 | 待接单→已接单→进行中→待验收→已完成 |
| 预约单 | 指定时间预约 | 待确认→已确认→待接单→进行中→已完成 |
| 车队单 | 多人组队 | 待组队→已组队→进行中→已完成 |
| 教学单 | 教学复盘 | 待开课→进行中→待评价→已完成 |

### 3.2 派单引擎升级

- **多维标签匹配**：游戏(30%)+段位(25%)+位置(20%)+声线(15%)+擅长(10%)加权计算
- **智能排序**：按标签契合度排序，取前10名在线打手推送
- **标签管理**：后台配置打手标签，支持游戏/位置/声线/段位/技能分类

### 3.3 竞价抢单模式

- **加急竞价**：加急订单开放竞价加价
- **原子锁防超抢**：Redis Lua脚本保证唯一性
- **竞价日志**：记录全部竞价记录，支持追溯

### 3.4 履约存证

- **服务计时**：订单进行中自动记录服务时长
- **凭证上传**：支持游戏录屏、战绩截图、其他凭证
- **OSS永久存储**：云端存储，纠纷仲裁直接调取

### 3.5 中途退单规则

- **后台可配置**：支持自定义时间阈值和退款比例
- **示例规则**：服务10分钟内全额退，超过30分钟按比例扣款
- **自动计算**：系统自动计算退款金额，无需人工核算

### 3.6 订单套餐

- **套餐类型**：小时套餐、多局套餐、教学套餐
- **统一标价**：后台配置套餐价格，简化下单流程
- **套餐展示**：小程序端套餐列表，游戏分类筛选

### 3.7 指定打手下单

- **收藏打手**：玩家可收藏心仪打手
- **预约服务**：预约指定时段专属服务
- **定时提醒**：系统定时推送接单提醒

## 八、技术架构优化

### 8.1 Swoole WebSocket服务

替代原Workerman FPM模式：
- **常驻进程**：支撑5000+打手同时在线长连接
- **心跳机制**：20秒心跳，60秒超时
- **掉线重连**：自动重连逻辑优化
- **房间管理**：支持多房间消息广播

### 8.2 MySQL分表方案

分表策略：
- **订单表**：按月分表（`order_archive_YYYYMM`）
- **聊天消息表**：按月分表（`chat_message_archive_YYYYMM`）
- **自动建表**：系统自动创建月度分表
- **查询路由**：根据时间自动路由到对应分表

### 8.3 Redis分层缓存

| 缓存层 | 存储内容 | 策略 |
|--------|----------|------|
| 永久缓存 | 热点打手数据、游戏列表、配置信息 | 永不过期，更新时主动失效 |
| 临时缓存 | 订单信息、会话数据 | 设置过期时间，自动淘汰 |

缓存防护：
- **击穿防护**：互斥锁 + 空值缓存
- **雪崩防护**：随机过期时间 + 分级失效
- **穿透防护**：布隆过滤器 + 参数校验

### 8.4 分布式锁统一封装

使用场景：
- 抢单、分账、提现、活体校验

特性：
- Redis SETNX + 过期时间实现
- 支持可重入
- 自动续期（看门狗机制）
- 标准化工具类，避免重复开发

### 8.5 接口限流中间件

分级QPS限制：
- 普通用户：60次/分钟
- 打手：120次/分钟
- 管理员：300次/分钟

## 十二、核心业务流程图

### 12.1 订单生命周期

```
创建订单 → 匹配派单/竞价抢单 → 打手接单 → 服务计时开始 → 服务进行中
    ↓                                   ↓
申请仲裁 → 仲裁举证 → 判责处理        履约凭证上传
    ↓
退款/赔付 → 反向分账回滚
```

### 12.2 资金流转

```
用户支付 → 监管账户 → 订单完成 → 分账指令 → 各角色账户
                                   ↓
                              提现申请 → 个税扣除 → 打款
                                   ↓
                              退款申请 → 反向分账 → 退款到用户
```

### 12.3 风控拦截

```
用户操作 → 实名校验 → 宵禁校验 → 消费限额校验 → 代练关键词检测 → 飞单关键词检测 → 执行操作
```
